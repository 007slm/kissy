Grid.Store

数据缓冲类

defined in： store.js

Class

  • Grid.Store (config) 数据缓冲类，缓存数据在浏览器中

Attributes

  • autoLoad {} 创建对象时是否自动加载
  • dataType {} 加载数据时，返回的格式,目前只支持"json、jsonp"格式
  • matchFunction {} 对象的匹配函数，验证两个对象是否相当
  • params {} 自定义参数，用于加载数据时发送到后台
  • proxy {} 连接信息，包含2个字段:
    url : 加载数据的地址
    method : 加载数据的方式"get","post"，默认值为"post"
    memoryData : {Array} 内存中的数据，如果未设置url，而是设置了memeryData,则加载数据时将加载内存中的数据
  • remoteSort {} 是否后端排序，如果为后端排序，每次排序发送新请求，否则，直接前端排序
  • root {} 加载数据时，返回数据的根目录
  • sortInfo {} 排序信息
  • totalProperty {} 加载数据时，符合条件的数据总数，用于分页
  • url {String} 是字段 proxy.url的简写方式，可以直接写在配置信息中

Methods

  • acceptChanges () 接受数据改变，将缓存的修改、新增、删除的集合清除
  • add (data, noRepeat, match) 添加记录
  • clear () 清除数据,清空所有数据
  • compare (obj1, obj2, field, direction) store的比较函数
  • compareFunction (obj1, obj2)
  • contains (record, match) 验证是否存在指定记录
  • find (field, value) 查找记录，仅返回第一条
  • findAll (field, value) 查找记录，返回所有符合查询条件的记录
  • findByIndex (index) 根据索引查找记录
  • findIndexBy (target, func) 查找数据所在的索引位置,若不存在返回-1
  • findNextRecord (record) 获取下一条记录
  • getCount () 获取加载完的数据的数量
  • getDeletedRecords () 获取删除的数据
  • getModifiedRecords () 获取更改的数据
  • getNewRecords () 获取添加的数据
  • getResult () 获取加载完的数据
  • getTotalCount () 获取表格源数据的总数
  • load (params) 加载数据,若不提供参数时，按照上次请求的参数加载数据
  • remove (data, match) 删除记录触发 removerecords 事件.
  • setResult (data) 设置数据，在不自动加载数据时，可以自动填充数据，会触发 load事件
  • setValue (obj, field, value, isMatch) 设置记录的值 ，触发 updaterecord 事件
  • sort (field, direction) 排序，根据Store的配置进行，前端排序或发送请求重新加载数据 远程排序，触发load事件，前端排序触发localsort事件
  • update (obj, isMatch) 更新记录 ，触发 updaterecord 事件

Events

  • acceptchanges () 数据接受改变，所有增加、删除、修改的数据记录清空
  • addrecords (e) 当添加数据时触发该事件
  • beforeload () 当数据加载前
  • beforeProcessLoad (e) 发生在，beforeload和load中间，数据已经获取完成，但是还未触发load事件，用于获取返回的原始数据
  • exception (e) 加载数据发生异常时触发
  • load (e) 当数据加载完成后
  • localsort (e) 前端发生排序时触发
  • removerecords (e) 当删除数据是触发该事件
  • updaterecord (e) 当更新数据指定字段时触发该事件

Attributes Detail

  • autoLoad 创建对象时是否自动加载
    Default Value
    false
  • dataType 加载数据时，返回的格式,目前只支持"json、jsonp"格式
    Default Value
    "json"
  • matchFunction 对象的匹配函数，验证两个对象是否相当
    Default Value
    function(obj1,obj2){return obj1==obj2};
  • params 自定义参数，用于加载数据时发送到后台
    Example
    • var store = new Store({
		url :'data',
		autoLoad : true,
		params: {id:'124',type:1}//自定义参数
	});
  • proxy 连接信息，包含2个字段:
    url : 加载数据的地址
    method : 加载数据的方式"get","post"，默认值为"post"
    memoryData : {Array} 内存中的数据，如果未设置url，而是设置了memeryData,则加载数据时将加载内存中的数据
    Example
    • var store = new Store({
		autoLoad : true,
		proxy: {url : 'data.php', method: 'get' }//按照'name' 字段降序排序
	});
  • remoteSort 是否后端排序，如果为后端排序，每次排序发送新请求，否则，直接前端排序
    Default Value
    false
  • root 加载数据时，返回数据的根目录
    Example
    • '{"rows":[{"name":"abc"},{"name":"bcd"}],"results":100}'
    Default Value
    "rows"
  • sortInfo 排序信息
    Example
    • var store = new Store({
		url : 'data.php',
		autoLoad : true,
		sortInfo: { field: 'name', direction: 'DESC' }//按照'name' 字段降序排序
	});
  • totalProperty 加载数据时，符合条件的数据总数，用于分页
    Example
    • '{"rows":[{"name":"abc"},{"name":"bcd"}],"results":100}'
    Default Value
    "results"
  • url {String} 是字段 proxy.url的简写方式，可以直接写在配置信息中
